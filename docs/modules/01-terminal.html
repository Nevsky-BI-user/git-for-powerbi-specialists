Термінал і файли

Що таке термінал (командний рядок)

Git найчастіше керують текстовими командами. Спершу розберемось, де їх вводити.

Ціль урокурозрізняти GUI і CLI та знати, де вводити команди Git.
Термінал / командний рядок (CLI)
Текстове вікно, де ти друкуєш команду словами, тиснеш Enter — і комп'ютер її виконує, виводячи відповідь текстом.
Простими словами: замість того, щоб клікати мишкою по кнопках, ти пишеш коротку інструкцію текстом. Це швидше й точніше для багатьох задач.
GUI vs CLI
GUI — графічний інтерфейс (вікна, кнопки, миша). CLI — текстові команди.
Простими словами: GUI — як меню в ресторані з картинками; CLI — як сказати офіціанту точне замовлення словами. Git працює і так, і так, але команди дають повний контроль.
Спокійно: термінал не кусається. Найгірше, що може статися, — команда не спрацює і сама напише текстом, чому. Тут набагато важче щось зламати випадково, ніж здається на перший погляд.
bash
# знак $ або > — це «запрошення»: термінал чекає команду $ git --version git version 2.45.0
Запрошення (prompt)
Символ на початку рядка ($, > або шлях), після якого вводять команду.
Простими словами: це «курсор готовності». Сам символ $ друкувати не треба — він уже там.
Як відкрити термінал — покроково для повного новачка
Спосіб 1 — через Пуск: натисни клавішу Windows (кнопка «Пуск»), набери git bash і тисни Enter — Windows сама знайде застосунок Git Bash серед результатів пошуку.
Спосіб 2 — у Провіднику, прямо в потрібній папці: відкрий папку проєкту в Провіднику, клікни правою кнопкою миші по порожньому місцю (не по файлу) і обери «Open Git Bash here». У Windows 11 цей пункт іноді ховається під «Показати додаткові параметри». Перевага способу: термінал одразу відкривається саме в цій папці, без жодного cd.
Спосіб 3 — прямо у VS Code: відкрий VS Code і натисни Ctrl + ` (клавіша під Esc, зазвичай ліворуч зверху) — знизу з'явиться вбудований термінал; переконайся, що обрана оболонка саме Git Bash, а не PowerShell.
На macOS/Linux термінал відкривають через застосунок «Terminal». Термінал цього курсу — Git Bash: він ставиться разом із Git, і всі приклади в уроках набрано саме в ньому.
Відкрий Git Bash прямо зараз і тримай його відкритим — далі всі приклади цього модуля повторюватимеш саме в ньому.
Термінал і файли

Де я і що навколо: pwd, cd, ls

Три команди, з якими ти ніколи не загубишся у файловій системі.

Ціль урокувільно визначати, де ти у файловій системі, і переміщатись між папками.
pwd
Показує поточну папку — повний шлях до місця, де ти зараз перебуваєш.
Простими словами: «де я?». pwd = print working directory.
ls (у Windows CMD — dir)
Виводить список файлів і папок у поточній директорії.
Простими словами: «що тут лежить?».
cd <папка>
Переходить в іншу папку. cd .. — на рівень вгору, cd ~ — у домашню папку.
Простими словами: «зайти всередину» папки або «вийти назовні». cd = change directory.
Типова помилка новачків: спершу дивляться на ls і дивуються, що не бачать потрібного файлу — а насправді просто ще не зайшли в потрібну папку через cd. Спершу зайди, потім дивись.
Навіщо взагалі cd, якщо можна просто клікнути папку мишкою
Бо термінал завжди «стоїть» в одній конкретній папці — поточній директорії — і виконує команди саме там, незалежно від того, яке вікно відкрито в Провіднику.
Простими словами: клік мишкою в Провіднику ніяк не повідомляє терміналу, де ти — це два незалежні світи. Коли пізніше дійдеш до Git, кожна команда (git status, git add, git commit) діє лише на папку, в якій зараз стоїть термінал через cd, — не на ту, що відкрита в Power BI Desktop чи в Провіднику. Найчастіша помилка новачка: відкрив звіт у Power BI Desktop, а термінал запустив «звідкись узагалі» (наприклад, з ярлика на робочому столі) — і Git-команди мовчки працюють не з тим проєктом або одразу видають помилку «not a git repository». Правило на все життя з Git: перш ніж вводити будь-яку git-команду, спершу переконайся командою pwd, що стоїш саме в папці цього проєкту, і за потреби зроби cd.
bash
$ pwd /home/ivan $ ls projects Documents Downloads $ cd projects $ pwd /home/ivan/projects $ cd .. $ pwd /home/ivan
У Git Bash на Windows шлях виглядатиме інакше, наприклад /c/Users/Ivan — це нормально, /c/ означає диск C:. Коли зайдеш командою cd у папку свого PBIP-звіту, сам рядок запрошення теж покаже, де ти: наприклад, Ivan@DESKTOP MINGW64 ~/Documents/sales-report. Тобто не треба щоразу набирати pwd, аби перевірити своє місце, — досить прочитати запрошення. Після того як зробиш репозиторій (модуль 02), там ще з'явиться назва поточної гілки в дужках, напр. (main).
pwdПоказати поточну папку.
lsСписок вмісту поточної папки.
cd reportsЗайти в папку reports.
cd ..Піднятись на одну папку вгору.
cd ~Перейти в домашню папку.
Абсолютний vs відносний шлях
Абсолютний — повний шлях від кореня (/home/ivan/projects або C:\Users\Ivan). Відносний — від поточної папки (reports, ../data).
Простими словами: абсолютний — «повна адреса міста й вулиці»; відносний — «сусідній під'їзд від того місця, де стоїш».
Ці команди навігації працюють і у вбудованому терміналі VS Code (Ctrl + `). Практичний сценарій: зберіг звіт із Power BI Desktop у папку проєкту, відкрив цю папку у VS Code — термінал одразу стоїть у ній, і ls покаже файли звіту без жодного cd.

Термінал і файли

Створюємо папки й файли: mkdir, touch, echo

Перш ніж робити Git-репозиторій, треба вміти створити папку проєкту й файли в ній.

Ціль урокустворювати папки й файли та записувати текст у файл з термінала.
mkdir <назва>
Створює нову папку. mkdir = make directory.
Простими словами: «зроби папку». mkdir reports створює папку reports.
Подивись анімацію ще раз і зверни увагу: mkdir щойно зробив рівно те саме, що права кнопка миші → «Створити» → «Папка» в Провіднику. Просто без миші й швидше.
touch <файл>
Створює порожній файл. У Git Bash (термінал цього курсу) команда touch працює як є — PowerShell чи CMD у курсі не знадобляться.
Простими словами: «створи порожній файл» із заданою назвою.
echo "текст" > файл
Записує текст у файл. Один > перезаписує файл; два >> дописують у кінець.
Простими словами: echo — «вивести текст»; стрілка > перенаправляє цей текст не на екран, а у файл.
cat <файл>
Показує вміст файлу в терміналі.
Простими словами: «покажи, що всередині файлу».
Виконуй ці команди у зрозумілій папці, напр. спершу cd ~/Documents — тоді створену папку легко знайти у Провіднику Windows (Документи) і вибрати в Power BI Desktop при збереженні PBIP.
bash
$ mkdir pdp-report # створили папку $ cd pdp-report $ touch README.md # порожній файл $ echo "# Звіт PDP" > README.md # записали рядок (перезапис) $ echo "Автор: Іван" >> README.md # дописали рядок $ cat README.md # Звіт PDP Автор: Іван
mkdir nameСтворити папку.
touch fileСтворити порожній файл.
echo "txt" > fileЗаписати текст у файл (перезапис).
echo "txt" >> fileДописати текст у кінець файлу.
cat fileПоказати вміст файлу.
Так стартує реальний проєкт: mkdir sales-report — а далі в Power BI Desktop зберігаєш звіт у цю папку як PBIP-проєкт. Git-репозиторій у ній ти зробиш уже в модулі 02.

Термінал і файли

Копіювання, переміщення, видалення: cp, mv, rm

Останні три файлові команди, без яких не обійтись у щоденній роботі.

Ціль урокукопіювати, переміщати, перейменовувати й обережно видаляти файли та папки.
cp <що> <куди>
Копіює файл. Для папки з вмістом — cp -r. Оригінал лишається на місці.
Простими словами: «зроби копію». cp = copy.
mv <що> <куди>
Переміщає файл або папку. Та сама команда й ПЕРЕЙМЕНОВУЄ: mv old.txt new.txt.
Простими словами: «перенеси» або «перейменуй». mv = move. Оригінал зникає зі старого місця.
rm <файл>
Видаляє файл. Папку з вмістом — rm -r.
Простими словами: «видали». rm = remove.
bash
$ cp README.md backup/README.md # копія у папку backup $ mv draft.md notes.md # перейменування $ mv notes.md docs/ # переміщення в папку docs $ rm old.txt # видалити файл $ rm -r temp/ # видалити папку з усім вмістом
cp file dir/Скопіювати файл у папку.
cp -r dir1 dir2Скопіювати папку з вмістом.
mv file dir/Перемістити файл.
mv old newПерейменувати.
rm fileВидалити файл.
rm -r dirВидалити папку з вмістом.
Класична історія: замість rm -r report_old пишуть rm -r report — і за секунду зникає папка з активним звітом, без бекапу. Перед rm -r завжди роби ls і ще раз перевіряй назву.
rm не має кошика. Видалене термінальним rm не потрапляє в «Кошик» — відновити штатними засобами не можна. Перед rm -r двічі перевір шлях. У Git-репозиторії закомічені файли захищені історією: їх можна повернути навіть після rm (урок «reset · restore · revert»).

Термінал і файли

Дрібниці, що рятують новачка

Кілька звичок, які роблять роботу в терміналі швидкою й безпомилковою.

Ціль урокупрацювати швидко: автодоповнення, історія команд, шляхи і шаблони.
TabАвтодоповнення назви файлу/папки. Почни друкувати й натисни Tab.
↑ / ↓Гортання попередніх команд (історія) — не треба передруковувати.
clear (Ctrl+L)Очистити екран термінала.
Ctrl + CПерервати поточну команду, що «зависла».
Копіювати / вставитиУ більшості терміналів: Ctrl+Shift+C / Ctrl+Shift+V (у Git Bash — права кнопка миші).
Коли термінал «завис» і що тоді робити
Ознака зависання: після Enter новий рядок запрошення ($) не з'являється, а команда просто «висить» без відповіді. Для новачка найчастіша причина — випадково запущена команда, яка чекає ще одного введення, або довгий вивід, що відкрився в повноекранному режимі перегляду (таке трапляється й у Git — деякі команди з довгою історією так роблять). Термінал не зламався, він чекає. Спочатку натисни Ctrl + C — команда перерветься, і повернеться знайоме запрошення. Якщо не спрацювало (буває саме для повноекранного перегляду) — спробуй клавішу q (quit), а вже потім Ctrl + C. Закривати вікно термінала хрестиком — крайній варіант, лише коли нічого інше не допомагає.
Ctrl+C у терміналі — це не «Копіювати»
У більшості інших програм Ctrl+C копіює виділений текст. У терміналі Ctrl+C означає «перерви команду», яка зараз виконується. Тому для копіювання тексту в Git Bash використовують інше поєднання — Ctrl+Shift+C (див. таблицю вище). Якщо натиснути звичайний Ctrl+C, думаючи, що копіюєш рядок з виводу команди, — можна ненароком перервати саме ту команду, яка щойно почала виконуватись.
Спеціальні позначки шляху
. — поточна папка; .. — на рівень вгору; ~ — домашня папка; / — роздільник папок (на Windows у багатьох інструментах теж працює /).
Простими словами: це «скорочення-адреси». Напр. cd ../data = «вийди на рівень вгору й зайди в data».
Шаблон * (зірочка)
Замінює «будь-які символи». *.tmdl — усі файли з розширенням .tmdl.
Простими словами: «усе, що підходить під шаблон». Знадобиться у .gitignore та командах.
bash
$ cd pd↹ ← натисни Tab, термінал сам допише «pdp-report/» $ ls *.md # показати лише файли .md README.md
Якщо у назві папки є пробіл — бери шлях у лапки: cd "My Reports". Інакше термінал сприйме це як дві окремі назви.
Чому пробіл у назві папки ламає команду
Термінал за замовчуванням розбиває введений рядок на окремі частини по пробілу. Кожна частина після команди сприймається як окремий, самостійний аргумент.
Простими словами: для термінала Sales Report — це не одна назва папки, а два слова: «Sales» і «Report» окремо. Реальний приклад із Power BI: звіт часто зберігають у папку на кшталт Sales Report Q1. Спроба зайти в неї без лапок ламається:
bash
$ cd Sales Report Q1 bash: cd: too many arguments # термінал побачив три окремі аргументи $ cd "Sales Report Q1" # лапки кажуть: це одна назва цілком

Встановлення Git і перший запуск

Перед роботою Git треба встановити й один раз представитися.

Ціль урокувстановити Git, перевірити версію і представитися системі.

1. Встановити

Звідки взяти Git
Офіційний сайт git-scm.com — інсталятор для Windows, macOS, Linux. На Windows він приносить ще й «Git Bash» — зручний термінал.
Простими словами: завантаж і встанови, як звичайну програму, з налаштуваннями за замовчуванням.
Докладніше про встановлення та налаштування — у модулі 00, урок «Встановлюємо Git і Git Bash».

2. Перевірити

bash
$ git --version git version 2.45.0 # якщо бачиш версію — все встановлено

3. Представитися (один раз)

bash
$ git config --global user.name "Іван Коваленко" $ git config --global user.email "ivan@company.com"
Виконай обидві команди git config прямо зараз у своєму терміналі, підставивши власне ім'я та email замість прикладу — без цього кроку Git не дозволить зробити перший коміт.
Навіщо це
Ім'я та email підписують кожен твій коміт, щоб в історії було видно авторство.
Простими словами: Git «знайомиться» з тобою, щоб підписувати твої збереження. Детальніше — у модулі B, урок «Налаштування: config і .gitattributes».
У VS Code відкрий термінал через Ctrl + ` і вводь команди прямо там — зручно бачити файли проєкту й термінал поруч.
Термінал і файли — перевірка

Вправи: термінал і файли

Перевір базу перед переходом до Git. Обери відповідь — побачиш пояснення.

Ціль урокузакріпити базу термінала перед стартом Git.
Тепер твоя черга. Перш ніж тиснути на відповідь, спробуй подумки (а краще — у відкритому терміналі) прогнати команду власноруч. Помилишся — не страшно, побачиш пояснення і запам'ятаєш ще міцніше.
